# 网站性能优化方案

## 当前问题分析
1. **主要瓶颈**：gameData.json文件过大（3.3MB），首次加载慢
2. **次要问题**：所有数据一次性加载，浪费带宽
3. **改进空间**：可以实现按需加载，减少首屏加载时间

## 优化方案

### 方案A：JSON文件分割 + 按需加载（推荐）⭐

#### 优点
- ✅ 首屏加载时间大幅减少（从3.3MB降至约100KB）
- ✅ 按需加载，节省带宽
- ✅ 用户体验更好
- ✅ 实施简单，无需后端

#### 实施步骤
1. 运行分割脚本：
   ```bash
   python split_json.py
   ```

2. 生成的文件结构：
   ```
   data/
   ├── gameData_index.json      # 索引文件（~100KB）
   ├── gameData_nintendo.json   # 任天堂数据
   ├── gameData_sony.json       # 索尼数据
   ├── gameData_pc_android.json # PC及安卓数据
   ├── gameData_other.json      # 其他平台数据
   └── gameData_misc.json       # 未分类数据
   ```

3. 修改script.js，使用按需加载逻辑（参考script_optimized.js）

4. 测试并部署

#### 预期效果
- 首屏加载时间：从 3-5秒 降至 0.5-1秒
- 带宽节省：约70-80%
- 用户体验：显著提升

---

### 方案B：启用Gzip压缩

#### GitHub Pages配置
GitHub Pages默认已启用Gzip压缩，但可以优化：

1. 在仓库根目录创建 `.htaccess`（如果使用自定义域名）：
   ```apache
   <IfModule mod_deflate.c>
     AddOutputFilterByType DEFLATE application/json
     AddOutputFilterByType DEFLATE text/javascript
     AddOutputFilterByType DEFLATE text/css
   </IfModule>
   ```

2. 或者使用Cloudflare CDN（免费）：
   - 注册Cloudflare账号
   - 添加你的域名
   - 启用自动压缩和缓存

#### 预期效果
- JSON文件压缩率：约60-70%
- 3.3MB → 约1MB

---

### 方案C：使用CDN加速

#### 免费CDN选项
1. **jsDelivr**（推荐）
   - 将JSON文件托管到GitHub
   - 使用CDN链接：`https://cdn.jsdelivr.net/gh/ytcwd3/ytcwd3.github.io@main/gameData.json`
   - 全球加速，自动压缩

2. **Cloudflare Pages**
   - 免费，全球CDN
   - 自动HTTPS
   - 无限带宽

#### 实施步骤
修改script.js中的fetch URL：
```javascript
const res = await fetch('https://cdn.jsdelivr.net/gh/ytcwd3/ytcwd3.github.io@main/data/gameData_index.json');
```

### 方案D：图片优化（如果有）

如果网站有图片资源：
1. 使用WebP格式
2. 添加懒加载：
   ```html
   <img src="image.jpg" loading="lazy" alt="描述">
   ```
3. 使用图片CDN（如：imgix, Cloudinary）

---

### 方案E：代码压缩

1. 压缩CSS和JS文件：
   ```bash
   # 安装工具
   npm install -g terser csso-cli

   # 压缩JS
   terser script.js -o script.min.js -c -m

   # 压缩CSS
   csso style.css -o style.min.css
   ```

2. 在HTML中引用压缩版本：
   ```html
   <link rel="stylesheet" href="style.min.css">
   <script src="script.min.js"></script>
   ```
