## 使用流程

### 更新单个分类（推荐）

**场景**：只想更新NS游戏

```bash
# 1. 编辑对应的模板
打开 tools/模板_NS.xlsx
添加/修改游戏数据（只需填写分享码）

# 2. 运行转换工具
cd tools
python excel_to_json_simple.py

# 3. 在弹出界面中：
- 选择文件：模板_NS.xlsx
- 输出目录：../data
- 点击"开始转换"

# 4. 生成文件：data/gameData_NS.json

# 5. 提交更新
cd ..
git add data/gameData_NS.json
git commit -m "更新：添加NS游戏"
git push
```

## 字段说明

| 字段 | 说明 | 示例 | 必填 |
|------|------|------|------|
| name | 游戏名称 | 塞尔达传说 | ✅ |
| category | 主分类 | 任天堂 | ✅ |
| subCategory | 子分类（多个用逗号分隔） | NS,动作,冒险 | ⭕ |
| code | 提取码 | 8888 | ⭕ |
| unzipCode | 解压密码 | 123456 | ⭕ |
| quarkCode | 夸克分享码 | abc123 | ⭕ |
| baiduCode | 百度分享码 | 1def456 | ⭕ |
| thunderCode | 迅雷分享码 | xyz789 | ⭕ |
| updateDate | 更新日期 | 2026.3.11 | ⭕ |

**重要**：只填写分享码，不要填写完整链接！
- ✅ 正确：`abc123def456`
- ❌ 错误：`https://pan.quark.cn/s/abc123def456`

## 常见问题

### Q1：为什么有5个模板文件？
**A**：每个模板对应一个JSON文件，避免数据混淆和重复。

### Q2：模板里的示例数据要删除吗？
**A**：不需要。示例数据来自现有JSON，可以保留。如果不想要，删除即可。

### Q3：更新一个分类需要转换所有模板吗？
**A**：不需要！只转换你修改的那个模板。

### Q4：Sheet名称可以改吗？
**A**：可以，但要注意输出文件名会跟着变。建议保持默认名称。
